# Git / Expo / React Native: шпаргалка для проекта

## 1. Клонирование репозитория на компьютер

Через терминал:
git clone [https://github.com/Endruch/Kula.git](https://github.com/Endruch/Kula.git) cd Kula
Через VS Code:
File → Clone Repository → вставить URL → выбрать папку → открыть проект.

## 2. Установка зависимостей

После клонирования или при добавлении новых библиотек:
npm install
или
yarn install
Это создаст папку node_modules/. Её не пушим в GitHub.

## 3. Работа с нативными папками (Expo)

Если проект managed workflow, папок android/ и ios/ может не быть — это нормально.
Если нужны нативные файлы:
expo prebuild
Появятся android/ и ios/. Их можно добавлять в Git, если планируются изменения нативного кода.

## 4. Коммиты и пуш изменений

После изменений в коде:
git add .
git commit -m "описание изменений"
git push
Если создаёшь новую ветку:
git checkout -b имя_ветки
git push -u origin имя_ветки
Не пушь большие бинарные файлы (APK, node_modules/, сборки).

## 5. Получение новых файлов с GitHub

Если кто-то добавил файл (например, README) на GitHub:
git pull
После этого все новые файлы появятся в локальном проекте.

## 6. История версий

Смотреть коммиты:
git log --oneline
Переключиться на старую версию:
git checkout <hash-коммита>
Для внесения изменений на старой версии создаём ветку:
git checkout -b имя_ветки

## 7. Сборка приложения для Android

Expo managed workflow:
expo build:android
Или через новый EAS Build:
eas build --platform android
Генерируется APK/AAB — не пушим в GitHub, хранить отдельно.

## 8. Важные правила

* В GitHub храним только исходный код и конфиги, не храните большие бинарники.
* Перед клонированием/переносом: убедись, что все изменения закоммичены и запушены.
* .gitignore обычно содержит: node_modules/, временные файлы IDE, сборки и .DS_Store.
